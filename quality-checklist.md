# 质量检查清单

## 每个故事完成后的检查

### 1. 功能是否实现？
- [ ] 所有描述的功能都已实现
- [ ] 没有遗漏的功能

### 2. 功能是否能正常使用？
- [ ] 功能可以正常启动
- [ ] 功能可以正常结束
- [ ] 没有运行时错误

### 3. 功能是否有视觉反馈？
- [ ] 用户能看到操作结果
- [ ] 有适当的动画或效果
- [ ] 没有空白或缺失的元素

### 4. 功能是否有错误处理？
- [ ] 异常情况有处理
- [ ] 有错误提示
- [ ] 不会导致程序崩溃

### 5. 功能是否经过实际测试？
- [ ] 手动测试了功能
- [ ] 测试了边界情况
- [ ] 测试了异常情况

## 通过标准

**所有问题都必须回答"是"，才能标记为"通过"**。

如果有任何问题回答"否"，必须修复，直到所有问题都回答"是"。

---

## 可玩性检查（针对游戏项目）

### 视觉资源
- [ ] 角色有独立的图片（不是圆圈）
- [ ] 每种怪物有独立的图片（不是圆圈）
- [ ] 每张地图有背景图
- [ ] 装备有图片（可选）

### 操作反馈
- [ ] 点击按钮有视觉反馈
- [ ] 角色移动有动画
- [ ] 攻击有动画
- [ ] 受伤有视觉反馈

### 用户体验
- [ ] 游戏流畅，没有卡顿
- [ ] 界面友好，容易理解
- [ ] 操作流畅，没有延迟
- [ ] 音效（可选）

---

## 用户验收检查（最终阶段）

### 可玩性
- [ ] 游戏可以正常玩
- [ ] 有明确的目标
- [ ] 有清晰的进度反馈

### 完整性
- [ ] 所有系统都可用
- [ ] 没有明显的 bug
- [ ] 用户体验良好

### 满意度
- [ ] 用户愿意玩这个游戏
- [ ] 用户认为游戏有价值

---

## 使用说明

### 每个故事完成后
1. 回答"每个故事完成后的检查"（5 个问题）
2. 如果所有问题都回答"是"，标记为"通过"
3. 如果有任何问题回答"否"，修复问题，重新回答

### 视觉资源生成阶段（游戏项目）
1. 回答"视觉资源"检查（4 个问题）
2. 如果所有问题都回答"是"，进入下一阶段
3. 如果有任何问题回答"否"，生成资源，重新回答

### 最终阶段
1. 回答"用户验收检查"（3 个部分，共 10 个问题）
2. 如果所有问题都回答"是"，项目完成
3. 如果有任何问题回答"否"，修复问题，重新回答
